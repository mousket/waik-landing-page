# Task 5b-04 — Report Flow Entry Redesign
## Phase: 5b — Staff Experience Redesign
## Estimated Time: 1–2 hours
## Depends On: task-5b-01 (new dashboard), task-03e (state machine)

---

## Why This Task Exists

The current report flow entry — incident type selection — exists as a
separate page but is visually disconnected from the redesigned dashboard.
When a nurse taps "Report Incident" from the new Zone 2 hero, the
transition should feel seamless and intentional, not like switching apps.

This task ensures the incident type selection screen (Screen 1 of the UI
spec) shares the same visual DNA as the new dashboard — dark teal header
with context, large tappable cards, consistent badge and color language.

It also adds the resident lookup component to the splash screen (Screen 2)
so the nurse can search for a resident by name or room number, which will
eventually pre-populate the residentId on the incident record.

---

## Success Criteria

- [ ] `npm run build` passes
- [ ] Incident type selection screen has dark teal header matching dashboard
- [ ] Header shows: "New Incident Report" + back arrow to dashboard
- [ ] 4 incident type cards: Fall, Medication Error, Resident Conflict, Wound/Injury
- [ ] Each card: icon, title, one-line description — minimum 80px height
- [ ] Tapping a card navigates to the splash screen for that type
- [ ] Splash screen (Screen 2) has resident search input at top
- [ ] Resident search returns results matching seed residents
- [ ] Selecting resident pre-fills resident name and room in splash form

# Task 5b-05 — UI Consistency Pass
## Phase: 5b — Staff Experience Redesign
## Estimated Time: 2–3 hours
## Depends On: task-5b-01 through task-5b-04 (all complete)

---

## Why This Task Exists

Four tasks have built new components and redesigned existing pages.
This task does a visual audit across the entire application to ensure:

1. The shared components in `components/shared/` are used consistently
   everywhere they appear — not mixed with older inline implementations
2. Design token values (colors, border radii, spacing) are sourced from
   `lib/design-tokens.ts` — not hardcoded strings
3. The staff and admin experiences feel like one product, not two codebases
   that happen to share a URL structure

This is also the task that adds the Residents tab to the staff bottom
tab bar and the resident search shortcut to the staff experience.

---
